# .consilium/

Local configuration directory for [Consilium](https://github.com/satoshi-uozumi/consilium) — a composable specialist gateway for Claude Code.

This directory is gitignored. It is generated by `consilium install` and updated by `consilium start`.

---

## Directory layout

```
.consilium/
  config.json          # Specialist registry
  gateway.json         # Gateway process config (server mode only)
  README.md            # This file
  specialists/         # Local specialist definitions
    <name>/
      SKILL.md         # Specialist instructions and capabilities
  plans/               # Feature plans written by /cs:plan
    <feature-name>/
      plan.md
```

---

## config.json schema

Lists all specialists. Each entry is one of:

```json
{
  "specialists": [
    { "name": "typescript" },
    { "name": "security", "url": "http://localhost:4000/security" },
    { "name": "infra",    "url": "http://remote-host:4000/infra" }
  ]
}
```

| Entry | Behaviour |
|---|---|
| `{ "name": "..." }` | Local — sub-agents read SKILL.md directly, no gateway |
| `{ "name": "...", "url": "http://localhost/..." }` | Gateway-served — MCP access via local gateway |
| `{ "name": "...", "url": "http://remote/..." }` | Remote — registered in `settings.json`, remote gateway serves it |

---

## gateway.json schema (server mode only)

Only needed when running a local gateway. All fields optional.

```jsonc
{
  "port": 4000,                              // Gateway port (default 4000; overrides $PORT)
  "specialistsDir": ".consilium/specialists", // Where to find specialist SKILL.md files
  "auth": {
    "issuer": "https://my-idp.example.com",   // OIDC issuer — enables Bearer token validation
    "audience": "consilium"
  }
}
```

## Adding a local specialist

1. Create a directory under `specialists/`:

   ```
   .consilium/specialists/<name>/SKILL.md
   ```

2. Write `SKILL.md` — freeform markdown describing the specialist's domain, capabilities, and any instructions for how Claude should consult it.

3. Add an entry to `config.json`:

   ```json
   { "name": "<name>" }
   ```

   No URL needed for local-only use. Sub-agents read SKILL.md directly.

---

## Adding a remote specialist

Add an entry with a URL to `config.json`:

```json
{ "name": "security", "url": "https://my-host/security" }
```

Run `consilium start` to register it in `.claude/settings.json`.

---

## Slash commands

Slash commands are installed to `.claude/commands/cs/` and available as `/cs:<name>` in Claude Code:

| Command | Purpose |
|---------|---------|
| `/cs:plan` | Consult specialists and produce a feature plan |
| `/cs:review` | Review the plan with specialists before implementation |
| `/cs:work` | Execute tasks from the plan |
| `/cs:done` | Archive a completed plan |

Plans are stored in `.consilium/plans/<feature-name>/plan.md`.

---

## Authentication (server mode only)

Set `auth.issuer` and `auth.audience` in `gateway.json` to enable Bearer token validation via OIDC. The gateway discovers the JWKS from the issuer's `/.well-known/openid-configuration`. Omit `auth` entirely for local use.
