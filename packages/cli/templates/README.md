# .consilium/

Local configuration directory for [Consilium](https://github.com/satoshi-uozumi/consilium) — a composable specialist gateway for Claude Code.

This directory is gitignored. It is generated by `consilium install` and updated by `consilium start`.

---

## Directory layout

```
.consilium/
  config.json          # Gateway configuration
  README.md            # This file
  specialists/         # Local specialist definitions
    <name>/
      SKILL.md         # Specialist instructions and capabilities
  plans/               # Feature plans written by /cs:consult
    <feature-name>/
      plan.md
```

---

## config.json schema

All fields are optional.

```jsonc
{
  "port": 4000,                        // Gateway port (default 4000; overrides $PORT)
  "local": {
    "specialistsDir": ".consilium/specialists",  // Where to look for local specialists
    "specialists": ["typescript"]      // Explicit list; omit to auto-discover all subdirs
  },
  "remote": [
    { "name": "security", "url": "https://my-host/security" }  // Remote MCP specialists
  ],
  "auth": {
    "issuer": "https://my-idp.example.com",   // OIDC issuer — enables Bearer token validation
    "audience": "consilium"
  }
}
```

`consilium start` reads this file, starts the gateway, and writes MCP entries into `.claude/settings.json` for each active specialist.

## Adding a local specialist

1. Create a directory under `specialists/`:

   ```
   .consilium/specialists/<name>/SKILL.md
   ```

2. Write `SKILL.md` — freeform markdown describing the specialist's domain, capabilities, and any instructions for how Claude should consult it. See the bundled `typescript` specialist for reference.

3. Run `consilium start` (or restart if already running). The new specialist is auto-discovered and registered as `consilium-<name>` in `.claude/settings.json`.

If `config.json` has an explicit `local.specialists` list, add the new name to it or remove the list to switch back to auto-discovery.

---

## Adding a remote specialist

Add an entry to `config.json`:

```json
"remote": [
  { "name": "security", "url": "https://my-host/security" }
]
```

The URL must point to a Consilium-compatible MCP endpoint (StreamableHTTP, exposing a `get_skill` tool). Run `consilium start` to pick up the change.

## Slash commands

Slash commands are installed to `.claude/commands/cs/` and available as `/cs:<name>` in Claude Code:

| Command | Purpose |
|---------|---------|
| `/cs:consult` | Consult specialists and produce a feature plan |
| `/cs:review` | Review the plan with specialists before implementation |
| `/cs:work` | Execute tasks from the plan |
| `/cs:done` | Archive a completed plan |

Plans are stored in `.consilium/plans/<feature-name>/plan.md`.

---

## Authentication (optional)

Set `auth.issuer` and `auth.audience` in `config.json` to enable Bearer token validation via OIDC. The gateway discovers the JWKS from the issuer's `/.well-known/openid-configuration`. Omit `auth` entirely for local use.
